## 1) 총평 (5줄)

1. **Value Statement 정합성: ✓ 양호** — "빠른 캡처→triage→오늘 일정 전환 + ADHD 친화"라는 핵심 목적을 20개 제안 전반이 일관되게 따르고 있음.
2. **P0 선정은 대체로 적합하나, "Undo/안전장치" 누락이 치명적** — 기존 030 계획에서 P0이었던 Undo가 이번 제안서에서 빠지면서 ADHD 사용자의 "실수 공포" 제거 핵심 축이 사라짐.
3. **ROI 측정 지표가 대부분 "관측 불가"** — 제안된 지표(예: triage 세션 중 ESC 시도 후 이탈률)는 현재 계측 인프라 없이는 측정 불가능; 달성 기준이 정의되지 않아 성공/실패 판단 모호.
4. **구현 범위 규칙 대부분 준수, 단 "세션 상태 저장" 표현이 애매** — "세션 단위 우선"이라고 하면서도 Dexie systemState 확장 가능성을 열어둬, localStorage 금지 규칙과 충돌할 여지 있음.
5. **접근성(A11y) 항목이 P1으로 밀려 있어 기본 키보드/SR 지원이 지연될 수 있음** — triage가 키보드 중심인데, ARIA 역할/포커스 링이 P1이면 핵심 사용자 경험이 반쪽.

## 2) 가장 위험한 가정 5개

| # | 위험 가정 | 왜 위험한가 | 검증 방법/대안 |
|---|---|---|---|
| 1 | **"단축키만 추가하면 진입 마찰이 해소된다"** | 단축키 발견성(#16)이 P0인데, 정작 단축키 학습 비용을 측정할 계측이 없음. 사용자가 단축키를 모르면 병목은 그대로. | 온보딩/힌트 노출 후 실제 사용률 A/B 테스트 필요 |
| 2 | **"추천 슬롯이 실패하면 대안을 주면 된다(#3,#4)"** | slotFinder 실패 빈도/원인 데이터가 없어 대안 UI가 자주 노출되면 오히려 "또 실패" 인식 강화. | slotFinder 실패 로그 분석 선행 필요 |
| 3 | **"세션 단위 상태(triage 목표/핀)는 저장 안 해도 된다"** | ADHD 사용자는 "잠시 나갔다 돌아오는" 패턴이 잦아, 세션 리셋 시 진행률 손실이 동기 저하 요인이 될 수 있음. | 짧은 세션 보존(5분) 또는 명시적 "세션 끝내기" 버튼 제공 |
| 4 | **"모달 vs 패널 이원화가 유지보수 부담만 올린다"(리스크 #3)** | 이 리스크를 인지했지만 대책이 "공통 컴포넌트"뿐. 패널 도입 시 진입점 분기, 상태 동기화, 테스트 커버리지 증가가 P1 범위를 초과할 가능성 높음. | 패널은 읽기 전용(view-only)으로 시작하거나, P2로 재조정 |
| 5 | **"명령 팔레트(#20)가 단축키 암기 대안"** | 명령 팔레트 자체가 High/High ROI로 P2인데, "단축키를 못 외우는 사용자" 문제를 해결하려면 P2가 너무 늦음. | 팔레트 MVP(인박스 열기/triage 토글만)를 P1으로 승격하거나, 인라인 액션 메뉴로 대체 |

## 4) 추가 제안 3개 (기존 20개 중 교체 후보)

> 아래 3개는 기존 20개 중 ROI가 낮거나 범위 초과인 항목과 교체를 권장

### 추가 A: **Undo 기반 안전장치(마지막 1회) + 스낵바 표준**
- **교체 대상**: #10 '보류/핀(세션)' (P2, Med/Low) — 보류보다 Undo가 ADHD 친화 핵심
- **근거**: 030 계획에서 P0였음. 삭제/이동 실수 복구 없이 triage 사용은 "무서워서 못 씀".
- **지표**: 삭제/이동 후 Undo 사용률 ≥20% (2주 후)

### 추가 B: **인박스 진입 시 자동 포커스 + triage 첫 항목 선택**
- **교체 대상**: #8 '빠른 캡처 미니 입력' (P2, Med/Med) — 캡처보다 triage 시작 마찰이 더 큼
- **근거**: 모달 열자마자 첫 항목이 선택되어야 키보드 triage가 바로 시작됨. 현재 분석에서 "열자마자 입력" 언급은 있으나 "열자마자 triage 선택" 부재.
- **지표**: triage 시작까지 키스트로크 수 ≤1

### 추가 C: **배치 완료 후 인라인 확인 메시지(토스트 대체)**
- **교체 대상**: #9 '템플릿 기반 블록 변환' (P2, Med/Med) — 템플릿보다 피드백 루프가 ADHD에 더 급함
- **근거**: #14 "토스트 의존도 축소"와 결합. 배치 성공 시 인박스 리스트 내에 "→오늘 14:00 배치됨" 인라인 메시지 1초 표시.
- **지표**: 토스트 발생 횟수/세션 ↓ 50%

## 7) Recommendations

1. **Undo를 P0에 즉시 추가** — 가장 시급. 이 없이 triage 채택률 예측 불가.
2. **ARIA 역할/포커스 링을 P0로 승격** — 기능이 아니라 작동 조건.
3. **#3+#4 통합, #16→#11 흡수** — 중복 UI/구현 방지.
4. **지표를 "채택률/성공률" 기반으로 재정의** — 관측 가능하고 달성 기준 명확하게.
5. **Target Release를 183~185 분할** — P0만 183, P1은 184~185.
