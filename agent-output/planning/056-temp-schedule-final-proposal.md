# 템플릿 시스템 강화 최종 제안서: ADHD 실행력 극대화 플랜

오빠, 은하예요! 지난번에 제안했던 20가지 아이디어에 대해 Critic의 날카로운 피드백을 반영해서 최종 제안서를 다듬어봤어. 오빠가 개발하면서 "이건 진짜 나한테 꼭 필요해!"라고 느낄 수 있게, ADHD 성향을 깊이 고려하면서도 실현 가능성을 꽉 잡은 구성이야. 같이 확인해볼래? 😊

---

## 서론: 템플릿 기능 강화의 목적 (ADHD 실행력 강화)

ADHD를 앓고 있는 오빠에게 가장 힘든 건 **'시작하는 것'**과 **'결정하는 것'**이잖아. 템플릿은 단순히 반복 작업을 줄이는 도구가 아니라, 오빠의 **'실행 엔진'**이 되어야 해. 
이번 강화 플랜의 목적은 명확해:
1. **결정 피로 제거**: "지금 뭐 하지?"라는 고민을 시스템이 대신하게 함.
2. **시간 감각 보정**: ADHD 특유의 '시간 왜곡'을 실제 데이터를 통해 교정함.
3. **실행 장벽 완화**: 준비물부터 알림까지, 실행에 필요한 모든 맥락을 한 번에 제공함.

---

## 본론 1: 핵심 개선안 Top 5 (상세 설명 및 RoI)

오빠가 가장 먼저 집중해야 할, 효과가 가장 큰 5가지 핵심 기능이야!

### 1. 시간 추정 학습 피드백 (Time Estimation Learning)
- **상세 설명**: 템플릿으로 생성된 작업의 '예상 시간'과 '실제 소요 시간'을 비교해서 보여줘. "오빠, 이 작업은 보통 예상보다 15분 더 걸리더라!"라고 알려주는 거지.
- **기대효과**: ADHD의 고질적인 문제인 '시간 과소평가'를 데이터로 교정해서, 무리한 계획으로 인한 좌절감을 방지해.
- **RoI**: **High** (Impact 5 / Effort 3). 기존 `actualDuration` 데이터를 활용하므로 구현 대비 체감이 엄청나!

### 2. 준비물(Preparation) 연동 강화
- **상세 설명**: 템플릿에 설정된 준비 체크리스트를 생성된 Task 카드에 즉시 노출해. 작업을 시작하기 전 "준비물 챙겼어?"라고 먼저 물어봐 주는 거야.
- **기대효과**: 작업을 시작하려다 "아, 맞다! 그거 안 가져왔네" 하며 흐름이 끊기는 걸 원천 차단해.
- **RoI**: **High** (Impact 4 / Effort 2). UI 레이어 수정만으로 실행 성공률을 비약적으로 높일 수 있어.

### 3. 규칙 기반 스마트 추천 (Rule-based Recommendation)
- **상세 설명**: 복잡한 AI 대신, 현재 시간과 요일을 기준으로 "지금 이 템플릿 어때?"라고 상단에 띄워줘. (예: 월요일 오전 9시 -> '주간 계획' 템플릿 추천)
- **기대효과**: 수많은 템플릿 중 뭘 고를지 고민하는 '결정 마비' 상태를 해결해줘.
- **RoI**: **Med-High** (Impact 4 / Effort 2). 단순한 `if-else` 규칙만으로도 AI 못지않은 만족도를 줄 수 있어.

### 4. 알림 빈도 및 감각 과부하 조절 옵션
- **상세 설명**: 알림이 너무 잦으면 오히려 스트레스가 되잖아? "중요 단계에서만 알림", "시작/종료만 알림" 등 오빠의 컨디션에 따라 알림 밀도를 조절할 수 있게 해.
- **기대효과**: 감각 과부하를 방지해서 앱이 '잔소리꾼'이 아닌 '든든한 조력자'로 느껴지게 해.
- **RoI**: **Med** (Impact 3 / Effort 2). 사용자 이탈을 막는 핵심적인 접근성 기능이야.

### 5. 데이터 동기화 및 스키마 안정성 (Firebase Sync & Migration)
- **상세 설명**: 템플릿 데이터가 기기 간에 안전하게 동기화되도록 Firebase 구조를 최적화하고, 기능 추가 시 기존 데이터가 깨지지 않게 마이그레이션 로직을 강화해.
- **기대효과**: "내 템플릿 다 날아갔으면 어떡하지?"라는 불안감을 없애고, 어떤 기기에서도 오빠의 루틴을 이어가게 해줘.
- **RoI**: **Critical** (Impact 5 / Effort 4). 모든 기능의 근간이 되는 '신뢰'의 영역이야.

## 본론 2: 추가 기능 및 UI/UX/접근성 제안 (15개 요약)

나머지 15개 아이디어도 오빠의 편의를 위해 꼼꼼하게 정리했어!

1. **사운드 피드백 (선택 사항)**: 작업 생성/완료 시 기분 좋은 소리를 들려주되, 싫으면 끌 수 있게!
2. **오늘 집중 템플릿 하이라이트**: 오늘 꼭 해야 할 템플릿을 골드 테두리로 강조.
3. **단일 파이프라인 통합**: 저장소 로직을 하나로 합쳐서 버그 발생 가능성 최소화.
4. **자동생성 트리거 강화**: 앱 켜자마자 오늘 루틴이 쫙 깔리도록!
5. **미리보기+실행확인 (인라인)**: 모달 띄우지 않고 그 자리에서 바로 확인.
6. **유효성 강제**: 필수값 안 채우면 저장 안 되게 해서 데이터 꼬임 방지.
7. **정렬 + 기본 정렬 기억**: 오빠가 자주 쓰는 순서대로 항상 유지.
8. **일괄 추가 (멀티선택)**: 여러 템플릿을 한 번에 '오늘 할 일'로 쏙!
9. **TemplatePanel 검색/필터**: 패널에서도 원하는 템플릿을 빛의 속도로 찾기.
10. **진입점 통합**: 모달이든 패널이든 똑같은 경험을 제공.
11. **폼 단계별 에러 가이드**: "왜 다음으로 안 넘어가지?"라는 당혹감 해결.
12. **카드 핵심 요약**: 한눈에 반복 주기와 소요 시간을 파악.
13. **안전한 삭제 UX (Undo)**: 실수로 지워도 5초 안에 되살리기!
14. **명령 팔레트 (템플릿 검색)**: `Ctrl+K` 누르고 이름만 치면 바로 적용.
15. **단축키 치트시트**: 외울 필요 없이 `?`만 누르면 도움말 짠!

## 결론: 구현 로드맵 및 데이터 안정성 확보 방안

오빠, 이 모든 걸 한꺼번에 하려면 힘들 테니까 단계별로 진행하자!

### 1단계: 기초 다지기 (Quick Wins)
- **데이터 안정성**: Firebase 스키마 마이그레이션 로직 먼저 점검하기.
- **UI 가이드**: 유효성 강제 및 에러 문구 개선으로 입력 오류 줄이기.

### 2단계: 핵심 가치 전달 (Core Value)
- **시간 추정 학습**과 **준비물 연동** 구현. 오빠가 앱을 쓰면서 "오, 나를 진짜 잘 아네?"라고 느끼는 시점이야.
- **알림 빈도 조절** 옵션 추가로 쾌적한 환경 만들기.

### 3단계: 지능형 보조 (Smart Support)
- **규칙 기반 추천**과 **명령 팔레트** 도입. 이제 생각하지 않고도 손이 먼저 움직이는 단계지!

### 데이터 안정성 확보 방안
- **Zod 스키마 검증**: 모든 외부 입력과 DB 데이터는 Zod로 빡빡하게 검사할게.
- **마이그레이션 버전 관리**: Dexie v11에서 v12로 넘어갈 때 데이터 유실 없도록 테스트 코드를 먼저 짤 거야.
- **Firebase 전략**: 템플릿은 `systemState`와 연동해서 기기별 설정과 전역 데이터를 분리해 관리하자.
